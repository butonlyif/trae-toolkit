# Daylog 机制

每次会话结束时，Agent 必须将本次会话的变更追加到项目根目录的 `DAYLOG.md`。

## 格式

```markdown
## YYYY-MM-DD (周X)

### 修复
- **简述**: 原因 → 方案

### 新增
- **简述**: 功能描述

### 架构
- **简述**: 决策 + 写入何处 (rules.md / design doc)

### 移植性
- **简述**: 路径/环境变量/平台相关修复
```

## 三类文档的粒度区分

| 文档 | 粒度 | 受众 |
|------|------|------|
| `DAYLOG.md` | 每日变更（Bug/新功能/重构） | 开发者自己回顾 |
| `CHANGELOG.md` | 版本级里程碑 | 用户/团队 |
| `.trae/rules.md` | 持久架构决策 | Agent + 新成员 |

## 守则

- 追加到文件末尾，不覆盖已有内容
- 一句话即可，不要长篇复述
- 如果当天已有日期标题，追加到该节下；否则新建日期节
- 作为会话最后一步执行，不打断正常开发流程
