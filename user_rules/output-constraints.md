# Agent 输出约束

> 每次输出前检查此文件。这些规则帮你省 Token、提效率。

## 禁止

- **禁止废话开头** — 不要说"好的我来帮你分析一下"、"让我看看这个问题"。直接给方案。
- **禁止重复需求** — 用户刚说的内容不要复述一遍。
- **禁止自动生成文档** — 不自动写 README、不自动写注释、不自动写 docstring。除非明确要求。
- **禁止自动写测试** — 除非明确要求。
- **禁止解释显而易见的内容** — 如果代码本身已经足够清晰，不额外解释。只解释"为什么这么做"而不解释"这段代码做了什么"。
- **禁止过度拆分** — 3 行重复代码 < 1 个过度抽象。不为"可能的未来需求"写代码。

## 必须

- **代码优先** — 先给出改动的代码，再给简短说明（如果需要）。
- **直接给方案** — 用户说"修 bug"，就给出修复后的代码，不要说"我分析了一下这个 bug 可能由以下原因导致..."
- **自检** — 每次输出前问自己：这段话删掉会影响用户理解吗？不会就删。
- **批量操作** — 多个独立工具调用合并到一个消息里。
- **更新 DAYLOG** — 每次输出结束时，自动将变更追加到 DAYLOG.md（一句话即可，不要长篇）。
