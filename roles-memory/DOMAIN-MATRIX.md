# 领域-角色能力矩阵

定义每个角色在不同领域中的核心能力和关注重点。

## 领域概览

| 领域 | 关键词 | 典型文件 |
|------|--------|---------|
| fpga | Verilog, VHDL, 时序收敛, 综合, 布局布线 | `.v`, `.sv`, `.vhd`, `.sdc`, `.xdc` |
| soc | 嵌入式, RISC-V, ARM, BSP, 驱动, RTOS | `.c`, `.h`, `.S`, `.ld`, Makefile |
| sdk | CLI, API, 工具链, 包管理, 扩展 | `.ts`, `.py`, `.go`, `.json`, `.yaml` |
| debug | 调试, 诊断, 日志, 性能分析 | 日志输出, 调试脚本, 性能报告 |
| vendor | 特定厂商工具, 专有流程, 兼容问题 | 厂商特定格式, 版本迁移 |
| aichip | AI加速器, 神经网络编译器, 算子 | MLIR, ONNX, 自定义IR |

## 角色能力矩阵

| 角色 | fpga | soc | sdk | debug | vendor | aichip |
|------|------|-----|-----|-------|--------|--------|
| **Builder** | RTL编写, IP集成, 约束编写 | C/ASM编写, BSP移植, 驱动开发 | CLI实现, API开发, 扩展开发 | 诊断工具开发 | 流程脚本, 适配代码 | 算子实现, 编译器后端 |
| **Optimizer** | 时序优化, 资源利用, 流水线 | 内存优化, 中断效率, DMA策略 | API设计, 性能优化, 代码质量 | 诊断效率, 信息完整度 | 工具链效率, 脚本质量 | 计算图优化, 内存规划 |
| **Tester** | 仿真验证, 时序检查, Lint | 单元测试, 集成测试, 上板验证 | CLI测试, API测试, 兼容性 | 日志验证, 场景覆盖 | 多版本测试, 多平台 | 精度验证, 性能基准 |
| **Docs** | 寄存器文档, 接口规范 | API文档, 移植指南, 驱动说明 | 用户手册, CLI参考 | 排错指南, FAQ | 安装指南, 兼容矩阵 | 架构文档, API文档 |
| **Researcher** | 新器件评估, IP对比 | 芯片选型, RTOS对比 | 技术选型, 工具对比 | 调试方法论 | 版本特性, 迁移路径 | 框架对比, 性能数据 |

## 领域交互矩阵

跨领域影响的常见路径：

| 源领域 → 目标领域 | 典型影响 |
|--------------------|---------|
| fpga → soc | RTL改寄存器 → BSP固件需同步 |
| soc → fpga | 驱动改内存映射 → RTL地址需对齐 |
| sdk → all | API/CLI 改接口 → 所有下游需适配 |
| vendor → fpga | 工具版本升级 → 约束/流程可能变化 |
| debug → sdk | 调试发现的设计问题 → SDK 需改进 |

## 使用方式

1. Builder 接到任务后，先查此表确认自己负责的领域
2. 跨领域改动时，检查源→目标的影响路径
3. Optimizer 审查时，对照此表检查跨领域影响
4. 新领域或新交互关系可追加到表中
